# ALS Alert System 
ALS, or amyotrophic lateral sclerosis, is a condition which results in loss of voluntary control
of muscles and, in later stages, an inability to speak. The goal of this project is to develop 
a low-cost eye-triggered alert system for persons with ALS to communicate a need for assistance
when their caregivers are not nearby

## Overview 
The ALS Alert system works by using Electrooculography (EOG) data. EOG is a method to measure
The electrical current generated by a patient moving their eyes from side to side. If a patient moves their eyes from side to side rapidly over a short time, they will trigger an alert. This alert will trip an alarm. attached to the microprocessor, and a notification will be sent to the caregiver's phone. Each  trigger is stored in Firebase, which handles authentication, storage, and app hosting. They are connected by WiFI. ## Repository Structure
1. Arduino code – Different versions exist in the folders Geronimo_Code and Microprosser_WithApp. These folders have .ino files that are put onto the Arduino device. 

2. Website – The index.html folder contains the markup for the website. Additionally, users can download the latest version of the Android App via an APK file. The website is currently hosted on https://nickslick03.github.io/ALS_Alert_Works/. If the link does not work, replace “nickslick03” in the URL with the username of the current owner of the GitHub repository. 

  - Releasing a new APK Version – To build a new APK to be released on the Github and on the website, you first need the project source code opened on Android Studio. Then, build an APK by following this tutorial (https://www.youtube.com/watch?v=3FujlwQoKuk). Then go to the GitHub repository page > releases > Draft a new release. Upload the APK as “als_alert.apk” and the “output-metadata.json” file. Give the release a version tag that’s newer than the previous release version tag (If it was previously v1.0.2, make it v.1.0.3 or v.1.1.0). Do NOT put beta in the tag, or it will not be downloadable from the GitHub pages website (but it will still be available on the GitHub releases page). 
  - Updating the Website – Simply update the index.html file, push the changes to master, and the gh pages should build and update the website. 
3. Android App – All other files and folders are part of the Android app. Download the Android Studio and follow the instructions on the Android app to work on the app.

## Installation

This repo contains two pieces of code that operate together. The bulk of the
repository contains the code for the phone app. This is written in Kotlin and Java.
In addition, we have provided the EOG code, written in Arduino C, that detects when
an alert has been triggered. ***In order for the app to operate, you must install BOTH***

## Standards
Since we are creating a medical device, there are several important codes and
standards that we need to be aware of to support the safety and well-being
of those we are trying to assist. ISO 60601 requirements govern commercial
medical devices. Although we are not intending to commercialize our final device,
this standard can aid us in initially designing our device in a way that could streamline
future commercialization.
